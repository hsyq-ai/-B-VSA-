## 2026-04-08 红智秘书「嵌入会话」UI 改造总结

### 一、背景与目标
- **场景**：员工端红智秘书页中，嵌入的会话窗（`/app/workspace-embed/...`）在「原生会话」卡片内展示。
- **主要问题**：
  - 输入框中的放大按钮只会放大输入框，不会放大整个会话区。
  - 嵌入会话区域底部附件上传 / 语音输入 / 放大按钮布局偏高，看起来“挤在”输入框内部。
  - 首条提示语（如「先从哪块开始？」）左侧存在被截断/裁切的问题。
  - 会话下方的「更多消息与上下文 0 条消息」区块在秘书场景中属于噪声，应隐藏。
  - 底部「懂你所需，伴你左右」免责声明在嵌入态中显得冗余。

---

### 二、功能行为改造：从“放大输入框”到“放大会话区”

**涉及文件**：
- `console/src/pages/Chat/index.tsx`
- `console/src/pages/Employee/Secretary.tsx`

#### 1. 会话内放大按钮改造
- 在聊天组件的 `sender` 配置中，通过 `morePrefixActions` 注入「放大会话区」按钮：
  - 使用 `Maximize2` 图标按钮，放在语音输入按钮右侧。
  - 仅在秘书嵌入态 (`isSecretaryEmbed`) 下显示，避免影响其他场景。
- 点击逻辑：
  - 不再操作输入框高度，而是调用 `requestSecretaryEmbedFocus`。
  - `requestSecretaryEmbedFocus` 内部通过 `postMessage` 向父窗口发送消息：`{ type: "copaw-embed-focus-request" }`。

#### 2. 父页面（秘书页）接收放大请求
- 在 `Secretary.tsx` 中监听 iframe 的 `message` 事件：
  - 收到 `copaw-embed-focus-request` 时：
    - 调用 `setChatFocused(true)` 进入「专注会话」模式。
    - 使用 `document.getElementById("secretary-chat-card")?.scrollIntoView(...)` 将「任务会话区」卡片滚动到视窗顶部附近。
- 对应 UI 行为：
  - 右侧部门看板等区域在 `chatFocused` 为 `true` 时收起，突出会话区。
  - iframe 高度随 `chatFocused` 动态调整（专注态下更高，近似工作区主窗口体验）。

### 三、嵌入会话布局与样式优化

**涉及文件**：
- `console/src/pages/Chat/index.module.less`
- `console/src/components/employee/ai/FloatingAiDock.tsx`
- `console/src/components/employee/ai/FloatingAiDock.module.less`

#### 1. 消息区左右裁切问题修复
- 针对 simple embed / secretary embed 的聊天气泡，做了如下样式调整：
  - 调整 `-bubble-list` 左右 padding，使内容不贴边，视觉更居中。
  - 将 `-bubble-content-wrapper` 的 `max-width` 设置为 `100%`，保证文本区域不会被内部横向滚动条“拖走”。
  - 原本对整块 `bubble-content` 使用横向滚动的策略拆解为：
    - 普通文本：不再横向滚动，支持自动换行（`overflow-wrap: anywhere`）。
    - 表格 / 代码块：单独控制滚动行为。

#### 2. 表格 & 代码块渲染策略
- 表格：
  - 将 `:global([class*="-bubble-content"] table)` 改为 `width: 100%`、`table-layout: fixed`，避免撑出横向滚动条，同时保证单元格内容可自动换行。
  - 对 `th` / `td` 设置 `white-space: normal` + `word-break: break-word`，确保长文案在单元格内自然断行，不再导致整体布局错位。
- 代码块 `pre`：
  - 仍保持 `overflow-x: auto`，方便查看长代码文本，但不会再影响整块消息的布局。

#### 3. 输入区底部按钮位置优化

- 目标：附件 / 语音输入 / 放大会话区 三个按钮从「输入框内部」视觉挤压，调整为更像「独立底栏」。
- 关键改动：
  - `chat-anywhere-input-wrapper`：
    - 减少底部 padding，使输入区本身在视觉上更“轻”。
  - `-sender-content-bottom`：
    - 改为带背景和上边框的底栏样式：
      - `margin: 14px -12px -12px`，整体向下延伸并拉满宽度。
      - 增加顶部边框和淡渐变背景，形成与输入区域明显区分的底栏。
      - `align-items: center`，按钮整体靠下对齐更自然。
  - `-sender-prefix` 与 `-sender-actions-list`：
    - 调整对齐方式和 padding，让按钮整体落在底栏中间偏下的位置，避免“浮在中腰部”的感觉。

#### 4. 上下文区与多余模块隐藏

- 浮动秘书入口组件 `FloatingAiDock` 中的「更多消息与上下文」区域：
  - 通过将 `.feedSection` 设置为 `display: none;`，在员工端隐藏该区块。
  - 不再显示「更多消息与上下文 0 条消息」相关文案，避免占用秘书浮窗有限空间。

#### 5. 嵌入态免责声明隐藏

- 默认配置中 `sender.disclaimer` 文案为 i18n 的「懂你所需，伴你左右」。
- 为简化秘书嵌入态界面：
  - 在 `senderEnhancements` 中，当 `isSimpleEmbed` 为 `true` 时，显式将 `disclaimer` 置空：`disclaimer: isSimpleEmbed ? "" : undefined`。
  - 在样式上也对 `-disclaimer` 做了 `display: none !important;` 保护，确保 simple embed 模式下底部不会再出现该提示条。
