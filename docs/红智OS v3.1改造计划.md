## 1. 当前系统概览（技术实现 + 已有能力）

### 1.1 总体架构
- **后端**：Python FastAPI（`copaw app`），提供聊天、推送、附件上传、文件下载等 API。
- **前端**：Vite/React Console，静态构建后由后端直接托管，前后端通过 HTTP 通信。
- **消息获取**：前端 Chat 页面轮询 `GET /api/messages/pull` 获取推送消息并创建会话。
- **工作目录**：`COPAW_WORKING_DIR=/home/featurize/work/aifscie/copaw_work`，所有数据落地到该目录。

### 1.2 核心模块与实现现状
- **用户与认证**
  - 账号数据：`auth.db`
  - JWT 登录态
- **会话与消息**
  - 会话数据：`copaw_work/sessions/`、`copaw_work/chats.json`
  - 推送队列：`copaw_work/push_messages.db`（统一消息存储，已替换内存队列）
  - 用户间消息推送会自动建立/保持双边会话，并持久化对话内容
- **通知/转发/管家式润色**
  - 支持“通知X”“请@X”等自然语言触发通知任务
  - 支持 LLM 将通知改写为“管家式通知”
  - 支持 LLM 识别“转告/回复”意图并回传给原发送人
- **附件上传与存储**
  - 上传接口：`/api/chat-files/upload`
  - 数据落库：`copaw_work/chat_files.db`
  - 文件落盘：`/home/featurize/work/aifscie/files/<username>/`
  - 上传日志：`copaw_work/upload_debug.log`
  - 附件清单：每个用户目录 `manifest.json` 记录 `upload_id ↔ 原始文件名 ↔ 用户 ↔ 时间`
- **附件解析与多模态**
  - 支持 PDF/DOCX/文本类文件正文抽取
  - 解析结果自动回写到同一消息文本块供模型使用
  - 图片/文件粘贴与拖拽上传已打通（含缩略预览与上传提示）
- **文件检索与关联**
  - 通过“找/查看/搜索某文件名”进行模糊检索
  - 优先当前登录用户的上传清单，找不到再全局扫描
  - 消息文本中自动注入 `[file:upload_id]` 便于追溯
- **事件与记忆沉淀（基础版）**
  - 事件日志：`copaw_work/events/YYYY-MM-DD.jsonl`
  - 员工日记：`copaw_work/memory/employees/<user_id>/daily/`
  - 组织日记：`copaw_work/memory/org/daily/`
  - 组织摘要/部门摘要：自动生成日报摘要文件

### 1.3 当前“已做到什么”（里程碑小结）
- 已实现“用户对用户通知 → 自动创建会话 → 双边持续对话 → 会话持久化”。
- 已实现“附件上传落库 + 落盘 + 解析进上下文”的稳定链路。
- 已实现“附件可追溯检索（manifest）+ 上传日志 + upload_id 融入对话”。
- 已实现“通知改写 + 回复转发”基本闭环。
- 已实现“组织与员工的事件记忆沉淀（基础版日志/摘要）”。

## 2. 与三份文档的对齐差距（宏观视角）

依据《系统前后端设计与实施方案》《深度报告》《v1.0发布说明书》，目前主要差距如下：
- **事件驱动架构**：当前为“文件事件 + 轮询”，尚未升级为真正的事件总线与多消费者组模型。
- **编排引擎**：缺少系统级的任务拆解/分发/聚合（DAG编排）机制。
- **混合路由大脑**：模型路由与复杂度分级尚未形成完整策略闭环。
- **Claw记忆体系**：目前只有“基础日志”，未形成“个人沙箱→脱敏同步→中央记忆”的闭环。
- **审计溯源与数字水印**：缺少完整的不可篡改链式追溯体系。
- **Skill生态**：尚未形成可安装/可授权/可版本化的技能商店与容器体系。
- **前端体验升级**：未形成 Chat + Canvas + 领导驾驶舱 等核心交互体系。
- **私有化与信创**：运行方式可私有化，但缺少完整的部署标准与可替换基础设施选型说明。

## 3. v3.1 改造待办与优化方向（完整清单）

### 3.1 事件驱动底座升级
- [ ] 统一“Markdown事件模型”，为所有输入/输出/技能调用建立可审计事件记录
- [ ] 事件总线落地：Redis Streams（轻量）/ Kafka（规模化）二级模式
- [ ] 支持事件订阅、重放、幂等与延迟重试
- [ ] 事件与会话上下文关联（session_id 与事件流联动）

### 3.2 调度与编排引擎（Lobster Orchestrator）
- [ ] 实现“意图解析→任务拆解→子任务分发→结果聚合”的编排流水线
- [ ] 建立 DAG 结构与编排状态机
- [ ] 支持层级式多Agent协作（跨部门/复杂任务）

### 3.3 认知与混合路由大脑
- [ ] 构建任务复杂度评分器
- [ ] 建立三层路由：低复杂度→本地小模型 / 中复杂度→本地大模型 / 高复杂度→私有化集群
- [ ] 管家式风格与合规对齐（价值观蒸馏）

### 3.4 Claw 数据封装与记忆系统
- [ ] 多模态数据采集 → 实体/关系抽取 → 时空戳/情感标签 → .openclaw 格式封装
- [ ] 个人沙箱记忆（隔离）+ 脱敏同步器（血脑屏障）+ 组织中央记忆（向上汇聚）
- [ ] 与组织日报/部门日报/项目记忆联动

### 3.5 审计溯源与安全体系
- [ ] 全链路审计字段标准化（时间戳、操作者、意图标签、推理依据、哈希校验）
- [ ] 关键输出数字水印
- [ ] 事件链式不可篡改日志
- [ ] “三不原则”落实：不碰数据、不设后门、不被绑架

### 3.6 Skill 生态与技能商店
- [ ] Skill 独立沙箱（Docker/WASM）与权限声明
- [ ] Skill 生命周期管理（注册/发布/审批/卸载/回滚）
- [ ] 首批核心 Skill 对齐四大功能矩阵：
  - [ ] 红色哨兵（领导层预警/合规雷达）
  - [ ] 任务协同中心（跨部门响应）
  - [ ] 红能党建（学习/合规护航）

### 3.7 前端体验升级（Chat + Canvas）
- [ ] 主工作台双栏：Chat + 动态画布
- [ ] 伴随式交互（指向性编辑/局部重写）
- [ ] 领导驾驶舱：组织拓扑、卡点高亮、全局透视

### 3.8 多渠道与组织集成
