# 书记驾驶舱真新闻接入实施指导

## 1. 背景与问题

当前“书记驾驶舱 -> 指示直达”页面里的“政策新闻栏目窗口”是展示型壳子：

- 前端使用 `defaultNewsWindows` 作为默认内容
- 页面再用 `/party/directive-center` 台账前几条记录覆盖卡片
- 后端没有真正的新闻采集、清洗、缓存、同步能力
- “同步数据 / 获取对标建议 / 生成配套文档”仍以演示交互为主

这会导致页面看起来像实时栏目，但实际展示的是静态文案或人工录入数据。

## 2. 改造目标

本次改造目标不是让前端直接抓外部网站，而是建立一条可交付的正式链路：

1. 后端定时采集官方站点最新内容
2. 清洗、去重、摘要后落库缓存
3. 前端直接读取系统内部接口展示栏目窗口
4. 支持书记手动触发“同步数据”
5. 支持把外部新闻一键“纳入指示台账”

## 3. 本轮实施范围（P0）

本轮先把“真数据替换假卡片”做起来，范围如下：

### 已纳入本轮的内容
- 新增新闻采集存储层
- 新增新闻同步服务
- 新增栏目窗口接口、栏目列表接口、手动同步接口、纳入台账接口
- 前端栏目窗口改为读取真实新闻接口
- 详情抽屉中展示栏目最近文章，并支持查看原文/纳入台账

### 暂不在本轮展开的内容
- 大模型自动生成高质量对标建议
- 多租户地方源管理后台
- 新闻源审计看板
- 同步失败告警与运维看板
- SASAC/地方单位的客户化深度适配

## 4. 数据流设计

```mermaid
flowchart LR
A[官方新闻列表页] --> B[后端新闻同步服务]
B --> C[链接提取/日期识别/正文摘要]
C --> D[SQLite缓存落库]
D --> E[/party/directive-news/windows]
D --> F[/party/directive-news/articles]
F --> G[书记驾驶舱栏目抽屉]
E --> H[书记驾驶舱栏目卡片]
F --> I[纳入指示台账]
I --> J[/party/directive-center]
```

## 5. 数据源策略

本轮采用“官方源优先、后端统一抓取、可配置覆盖”的方式。

### 默认源
1. `central`：中国政府网要闻
   - `https://www.gov.cn/yaowen/liebiao/`
2. `sasac`：中国政府网最新政策（P0 先作为监管政策窗口）
   - `https://www.gov.cn/zhengce/zuixin/`
3. `local`：北京市政府要闻（P0 作为地方示例源，可替换）
   - `https://www.beijing.gov.cn/ywdt/yaowen/`

### 可配置项
通过环境变量支持后续替换：

- `COPAW_DIRECTIVE_NEWS_CENTRAL_URL`
- `COPAW_DIRECTIVE_NEWS_POLICY_URL`
- `COPAW_DIRECTIVE_NEWS_LOCAL_URL`
- `COPAW_DIRECTIVE_NEWS_LOCAL_SOURCE_NAME`
- `COPAW_DIRECTIVE_NEWS_SYNC_INTERVAL_SECONDS`

说明：
- 第二个窗口在 P0 先使用“中国政府网最新政策”作为监管政策专窗，后续可替换为国务院国资委或客户指定监管源
- 第三个窗口默认使用北京要闻，只是提供一条可运行的地方窗口样例，交付时应按客户属地替换

## 6. 后端改造说明

### 新增文件
- `src/copaw/app/directive_news_store.py`
- `src/copaw/app/directive_news_service.py`

### 修改文件
- `src/copaw/app/routers/party.py`
- `src/copaw/app/_app.py`

### 后端职责拆分

#### 6.1 `directive_news_store.py`
负责：
- 建新闻缓存表
- 建同步状态表
- 文章 upsert
- 按栏目查询最新文章
- 查询栏目最近文章列表
- 记录最后一次同步状态

#### 6.2 `directive_news_service.py`
负责：
- 维护 3 个栏目源配置
- 抓取列表页
- 解析标题、链接、日期
- 抓取详情页并提取摘要
- 定时循环同步
- 手动强制同步
- 将文章转化为 `directive-center` 台账记录

#### 6.3 `party.py`
新增接口：
- `GET /party/directive-news/windows`
- `GET /party/directive-news/articles?channel=central&limit=10`
- `POST /party/directive-news/sync`
- `POST /party/directive-news/articles/{article_id}/promote`

#### 6.4 `_app.py`
在应用启动时：
- 初始化新闻同步服务
- 启动后台定时同步循环
- 把服务对象挂到 `app.state.directive_news_service`

## 8. 接口约定

### 8.1 获取栏目窗口
`GET /api/party/directive-news/windows`

返回结构示例：

```json
{
  "synced_at": "2026-04-08T11:20:00+08:00",
  "windows": [
    {
      "channel_key": "central",
      "article_id": "directive-news-xxx",
      "title": "...",
      "source": "中国政府网",
      "origin_url": "https://...",
      "published_at": "2026-04-08T08:00:00+08:00",
      "summary": "...",
      "digest": "...",
      "level": "国家级",
      "policy_type": "战略部署",
      "suggestion": "...",
      "document_label": "...",
      "sync_status": "success",
      "synced_at": "2026-04-08T11:20:00+08:00"
    }
  ]
}
```

### 8.2 获取栏目文章列表
`GET /api/party/directive-news/articles?channel=central&limit=10`

### 8.3 手动同步
`POST /api/party/directive-news/sync`

请求体示例：

```json
{
  "channel": "central",
  "force": true
}
```

### 8.4 纳入指示台账
`POST /api/party/directive-news/articles/{article_id}/promote`

## 9. 验收标准

### 功能验收
- 打开“指示直达”页面后，栏目卡片展示真实新闻标题、来源、发布时间
- 点击“同步数据”后，可触发服务端重新抓取
- 点击栏目详情，可看到该栏目最近多条真实文章
- 点击“查看原文”可打开官方原始链接
- 点击“纳入指示台账”后，`directive-center` 台账新增对应记录

### 稳定性验收
- 外部源失败时页面不崩溃
- 已同步过的数据可作为缓存继续展示
- 同步状态可返回到前端
- 不因某一个源失败影响其他栏目同步
