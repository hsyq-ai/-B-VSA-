# CoPaw 修改代码后的构建与启动

每次修改代码后，按以下步骤构建、部署并重启服务。

---

## 一、修改类型与对应操作

| 修改类型 | 需要执行的操作 |
|---------|----------------|
| **仅后端 Python** | 重启 CoPaw |
| **仅前端（console）** | 构建前端 → 重启 CoPaw |
| **后端 + 前端** | 构建前端 → 重启 CoPaw |
| **开发模式安装** | 首次或依赖变更时 `pip install -e .` |

---

## 二、前端构建

### 2.1 进入 console 目录

```bash
cd CoPaw/console
```

### 2.2 安装依赖（首次或 package.json 变更时）

```bash
npm install
```

### 2.3 构建

```bash
npm run build
```

**说明**：
- 构建产物直接输出到 `CoPaw/src/copaw/console/`（index.html、assets/）
- 由 `vite.config.ts` 的 `outDir` 配置，**无需手动复制**
- 后端会从该目录 serve 静态资源

### 2.4 前后端不同源时（可选）

若前端访问地址与后端 API 不同源，构建时指定 `BASE_URL`：

```bash
BASE_URL=http://后端地址:8088 npm run build
```

---

## 三、后端安装（开发模式）

本地开发时建议使用可编辑安装，代码改动后无需重新 `pip install`：

```bash
cd CoPaw
pip install -e .
```

依赖变更（如 `pyproject.toml` 修改）后需重新执行上述命令。

---

## 四、重启 CoPaw

### 4.1 停止现有进程

```bash
pkill -f "copaw app"
```

或手动找到进程 ID 后 `kill <pid>`。

### 4.2 启动

```bash
cd CoPaw
export COPAW_WORKING_DIR=/path/to/copaw_work   # 可选，默认 ~/.copaw
copaw app --host 0.0.0.0 --port 8088
```

### 4.2.1 HTTPS 启动（可选）

> 说明：浏览器麦克风/摄像头 API 需要 HTTPS。  
> **生产建议使用“可信证书”**（域名正确 + 受信 CA），自签名证书可能仍会被浏览器限制。

**生成自签名证书（开发测试）**：

```bash
cd CoPaw
mkdir -p certs
openssl req -x509 -newkey rsa:2048 \
  -keyout certs/key.pem -out certs/cert.pem \
  -days 365 -nodes -subj "/CN=dev"
```

**HTTPS 启动**：

```bash
copaw app --host 0.0.0.0 --port 8088 --https \
  --ssl-certfile certs/cert.pem \
  --ssl-keyfile certs/key.pem
```

> 若要真正通过浏览器权限校验，请使用域名 + 可信证书。  
> 可选方案：用反向代理（Caddy/Nginx）在 443 上托管 TLS，再反代到 8088。

**后台运行**：

```bash
nohup copaw app --host 0.0.0.0 --port 8088 >> /tmp/copaw.log 2>&1 &
```

### 4.3 沙箱环境说明（必须）

如果运行环境存在**沙箱隔离**（如 Codex/Featurize），在沙箱内启动
会导致外网无法访问，甚至本地 `127.0.0.1:8088` 也无法连通。

需要在**沙箱外**启动服务，并使用端口转发暴露公网访问：

```bash
# 沙箱外启动（示例）
cd /home/featurize/work/aifscie/CoPaw
export COPAW_WORKING_DIR=/home/featurize/work/aifscie/copaw_work
export COPAW_LOGIN_OUTBOX_DELIVERY=1
export PYTHONPATH=/home/featurize/work/aifscie/CoPaw/src
nohup /environment/miniconda3/bin/copaw app --host 0.0.0.0 --port 8088 >> /tmp/copaw.log 2>&1 &

# 暴露端口（示例）
featurize port export 8088
```

### 4.4 验证

```bash
curl -s http://127.0.0.1:8088/api/version
```

返回 `{"version":"x.x.x"}` 表示启动成功。

## 五、完整流程示例

**场景**：修改了前端 Chat 组件和后端 notification_agent

```bash
# 1. 构建前端
cd CoPaw/console
npm run build

# 2. 停止旧进程
pkill -f "copaw app"

# 3. 启动
cd ..
export COPAW_WORKING_DIR=/home/xxx/copaw_work
nohup copaw app --host 0.0.0.0 --port 8088 >> /tmp/copaw.log 2>&1 &

# 4. 验证
sleep 10
curl -s http://127.0.0.1:8088/api/version
```

## 六、常见问题

### 1. 前端改了但页面没更新

- 确认已执行 `npm run build`
- 确认构建产物在 `CoPaw/src/copaw/console/`（index.html、assets/）
- 清除浏览器缓存或强制刷新（Ctrl+Shift+R）

### 2. 后端改了但行为没变

- 确认已重启 `copaw app`
- 若使用 `pip install .` 非可编辑安装，需重新 `pip install .` 后再重启

### 3. 构建产物位置

- Vite 输出：`CoPaw/src/copaw/console/`
- 后端 `_resolve_console_static_dir()` 使用 `pkg_dir/console`，即该目录
- **无需手动复制**

### 4. 工作目录

- 默认：`~/.copaw`
- 自定义：`export COPAW_WORKING_DIR=/path/to/copaw_work`
- 配置文件、event_bus、auth.db 等均在此目录下
