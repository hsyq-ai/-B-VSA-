# 红智OS v1.0 前端改造方案

> 目标：将现有 Console 前端定位为“管理员后台”，新增“普通员工端”作为红智OS主交互界面，体现自然语言驱动与组织级记忆的产品理念。

## 0. 文档信息
- 版本：v1.0（草案）
- 日期：2026-03-19
- 适用范围：CoPaw 前端（console）

---

## 1. 改造目标与原则

### 1.1 目标
- 保留现有 Console 作为 **管理员后台**（运维、配置、模型、技能与系统管理）。
- 新增 **普通员工端** 前端界面，聚焦“工作台+对话+组织记忆”。
- 减少员工端复杂度，把交互入口收敛为自然语言与任务卡。

### 1.2 设计原则
- 自然语言即入口（NUI）
- Chat + Canvas 双栏工作台
- 事件可追溯、知识可沉淀
- 结构简洁、交互低负担
- 强调“管家式通知 + 任务闭环”

---

## 2. 当前前端定位与重构策略

### 2.1 现有前端（Console）现状
- 侧栏包含：Channels / Sessions / CronJobs / Heartbeat / Skills / Tools / MCP / Models / Security 等系统级模块。
- 适合系统管理员，但不适合普通员工。

### 2.2 重构策略
- **保留原 Console 为管理员后台**（推荐路由前缀 `/admin`）。
- **新增员工端应用入口**（推荐路由前缀 `/app`）。
- 两套前端共用登录与基础组件，但页面结构完全分离。

---

## 3. 员工端信息架构（IA）

建议员工端主导航只保留 5~7 个入口：

1. **工作台（Workspace）**
   - Chat + Canvas 双栏
   - 任务、通知、报告都从此入口展开
2. **会话中心（Sessions）**
   - 系统对话（用户 ↔ 系统）
   - 人际对话（用户 ↔ 用户）
3. **通知与待办（Inbox）**
   - 管家式通知
   - 待办/审批/提醒
4. **文件与知识（Files）**
   - 个人上传
   - 组织共享
5. **记忆与档案（Memory）**
   - 个人记忆
   - 组织记忆摘要
6. **个人档案（Profile）**
   - 员工信息、部门、岗位
7. **设置（Settings）**
   - 通知方式、语言、隐私

---

## 4. 员工端核心页面设计

### 4.1 工作台（Workspace）
- 左栏：对话流（支持文本/语音/附件/图片/粘贴）
- 右栏：动态画布（摘要、报告、任务卡、溯源报告）
- 支持“指向性编辑”：选中画布 → 指令改写

### 4.2 会话中心（Sessions）
- 明确区分两类会话：
  - 系统会话（用户 ↔ 系统）
  - 人际会话（用户 ↔ 用户）
- 会话列表展示：来源、摘要、未读状态

### 4.3 通知与待办（Inbox）
- 卡片式任务流 + 表格视图
- 状态流转同步到列表（待办/处理中/已完成）
- 支持“一键回复/一键完成/转交”

### 4.4 文件与知识（Files）
- 个人文件库 + 组织共享库
- 文件详情显示：上传时间、来源、关联会话、解析摘要
- 支持模糊检索与 upload_id 追溯

### 4.5 记忆与档案（Memory）
- 个人记忆：自动汇总工作日志
- 组织记忆：脱敏聚合摘要
- 支持日报/周报输出

### 4.6 个人档案（Profile）
- 姓名、部门、岗位、负责人
- 与记忆系统绑定

---

## 5. 功能取舍与角色隔离

### 5.1 员工端不展示（仅管理员后台保留）
- Channels
- Sessions（系统级）
- CronJobs
- Heartbeat
- Skills 管理
- Tools / MCP / Agent Config
- Models / Environments / Security / Token Usage
- Manager

### 5.2 员工端必须强化
- 管家式通知
- 任务闭环与状态流转
- 附件即上下文解析
- 可追溯信息链路

---

## 6. 员工端交互细节要求

- **聊天即入口**：所有操作优先走自然语言
- **新会话自动区分**：系统会话 vs 人际会话
- **附件即上下文**：即使无文本，也必须解析
- **状态可视化**：任务与通知在列表/表格中显式状态
- **溯源入口**：每条通知/报告可追溯来源

---

## 7. 路由规划建议

- `/admin/*`：现有 Console（管理员后台）
- `/app/*`：员工端前台

示例：
- `/app/workspace`
- `/app/sessions`
- `/app/inbox`
- `/app/files`
- `/app/memory`
- `/app/profile`
- `/app/settings`

---

## 8. 分阶段落地建议

**阶段1：结构拆分**
- 将现有 Console 独立为管理员后台
- 新建员工端布局与路由

**阶段2：工作台与会话中心**
- 先实现 Chat + Canvas 主工作台
- 分离系统会话与人际会话

**阶段3：通知与待办闭环**
- 状态流转可视化
- 一键回复/转交机制

**阶段4：文件与记忆模块**
- 文件库与组织记忆摘要
- 支持日报输出

---

## 9. 验收标准（v1.0 前端）
- 员工端交互入口收敛到“工作台 + 会话 + 通知 + 文件 + 记忆”。
- 管理员后台保留系统级配置与管理功能。
- 员工端不暴露运维类功能。
- 通知闭环、附件解析与状态流转可视化。
