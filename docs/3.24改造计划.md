# 3.24 改造计划（科研部能力升级，实验自动优先）

## 1. 改造目标

围绕高校老师、研究员、学生的科研场景，在现有 CoPaw 系统上完成一轮可落地升级：

1. 以“实验自动化”作为第一优先级，形成 `实验执行 -> 报错诊断 -> 修复建议 -> 复现实验脚本` 闭环。
2. 将科研规划从单轮问答升级为可追踪流程任务（选题、假设、方案、评估）。
3. 完整补齐科研工具链：知识检索、数据分析、科研创作、论文生成、业界跟踪。
4. 尽可能复用 AutoResearchClaw 代码；不能直接复用时采用同构设计与思路复用。

---

## 2. 当前系统基线（已具备）

基于现网代码，科研部已经具备以下基础：

1. 仪表盘菜单结构：`科研规划`、`科研工具`、`数字专家`。
2. 科研规划/科研工具的 8 个 dashboard 场景 trigger 已接入。
3. 数字专家导航已扁平化（不再经过“虚拟员工”二级）。
4. 专家中心已支持按部门展示候选专家，不再依赖默认（通用部门）兜底。

## 3. AutoResearchClaw 能力映射（对接 CoPaw）

### 3.1 科研规划（流程编排）

目标：把 `选题研判 / 质量评估 / 头脑风暴` 升级为“阶段化任务”。

映射方案：

1. 参考 AutoResearchClaw 的 stage pipeline 机制，为科研规划建立阶段状态机：
   - `topic_selection`（选题）
   - `hypothesis`（假设）
   - `method_plan`（方案）
   - `evaluation`（评估）
2. 每个阶段输出标准化工件：输入、结论、风险、下一步动作、负责人。
3. 在前端增加“流程进度条 + 阶段卡片”，支持断点续跑与回溯。

### 3.2 知识检索（可引用依据包）

目标：从“只给答案”升级为“证据化输出”。

映射方案：

1. 复用文献检索与聚合思路，形成统一 `Evidence Pack`：
   - 文献清单（标题、来源、年份、链接）
   - 核心结论（支持点）
   - 争议点（冲突证据）
2. 每条结论附带出处引用，避免不可追溯内容。
3. 输出格式固定，便于科研创作/论文生成直接引用。

### 3.3 数据分析（实验自动化优先）

目标：先打通实验自动闭环，是本次改造第一优先级。

映射方案：

1. 参考 `experiment_repair` 机制，支持：
   - 上传实验代码/日志/数据样例
   - 自动定位报错根因
   - 生成修复建议（代码级、参数级、环境级）
   - 生成复现实验脚本（含依赖与执行说明）
2. 建立“实验工单”实体，记录每次修复尝试的输入、变更与结果。
3. 每次修复后自动回归验证，输出“是否通过 + 差异说明”。

### 3.4 科研创作 / 论文生成（质量闸门）

目标：提升稿件一致性与引用可靠性。

映射方案：

1. 接入论文质量闸门（结构完整性、逻辑连贯性、方法可复现性）。
2. 接入引用校验（来源存在性、引用-结论一致性、疑似幻觉提示）。
3. 支持生成“修订建议报告”（问题项、严重级别、建议改法）。

### 3.5 业界跟踪（趋势雷达）

目标：形成持续更新的研究方向观察能力。

映射方案：

1. 构建方向热度看板（方向、热度、增长、代表论文/机构）。
2. 每周自动生成“机会雷达简报”：
   - 新兴方向
   - 值得复现实验
   - 可投稿窗口建议

## 4. 分阶段实施计划（3.24 起）

## 阶段 A（P0，1-2 周）：实验自动闭环 MVP

目标：科研部用户可上传实验资产并获得可执行修复方案。

交付项：

1. 新增实验工单与执行记录数据模型（任务、日志、修复建议、验证结果）。
2. 新增“实验诊断/修复”后端编排服务（先串通流程，再逐步增强模型效果）。
3. 数据分析菜单新增“实验自动化”入口页（上传 -> 诊断 -> 修复 -> 复现脚本）。
4. 数字专家中的 `实验管家`、`数据专员` 绑定实验工单能力。

验收标准：

1. 可上传最小实验包并返回结构化诊断结果。
2. 可生成至少一版修复建议与复现实验脚本。
3. 可记录修复前后结果并在页面展示状态流转。

## 阶段 B（P1，2-3 周）：规划流程化 + 检索证据化

目标：科研规划和知识检索从“问答工具”升级为“过程资产”。

交付项：

1. 科研规划引入阶段状态机与阶段工件存储。
2. 选题研判/质量评估/头脑风暴统一接入阶段任务框架。
3. 知识检索输出 `Evidence Pack` 标准格式，并支持导入科研创作。

验收标准：

1. 同一课题可看到阶段进度、阶段结论和历史版本。
2. 检索结果可直接为论文草稿生成“依据引用块”。

## 阶段 C（P2，2 周）：论文质量闸门 + 趋势雷达

目标：形成科研成果质量保障与方向洞察闭环。

交付项：

1. 论文生成链路加入质量闸门与引用校验。
2. 业界跟踪加入周报机制与机会雷达面板。
3. 关键流程打通：检索 -> 创作 -> 论文生成 -> 质量校验 -> 修订。

验收标准：

1. 论文草稿可输出质量评分与问题清单。
2. 周报可按科研方向自动汇总并可追溯来源。

1. 优先复用：流程阶段设计、实验修复循环、论文校验思路。

## 6. 风险与控制

1. 风险：实验执行环境差异大（依赖、GPU、数据权限）导致复现不稳定。  
   控制：先支持“受控最小运行时”，逐步扩容到复杂环境。
2. 风险：自动修复建议准确率不稳定。  
   控制：输出建议分级（高置信/待验证），并保留人工确认开关。
3. 风险：多模块同时上线影响体验。  
   控制：严格按 P0 -> P1 -> P2 灰度发布，每阶段可独立回滚。

## 8. 成功标志（业务口径）

1. 科研部用户从上传实验包到拿到可执行修复方案，单次链路可在一个工作会话内完成。
2. 科研规划任务可追踪、可续跑、可交接，不再依赖一次性对话。
3. 检索、创作、论文生成和质量校验实现可追溯闭环，形成可沉淀资产。
