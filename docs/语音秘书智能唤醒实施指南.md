# 语音秘书智能唤醒实施指南

> 版本：v3.9 | 日期：2026-04-13
> 目标：点击按钮唤醒语音秘书，唤醒后能智能判断用户是否在对其说话，不说话时不乱回

---

## 1. 现状问题

当前语音秘书的交互链路：

```
前端麦克风 → ScriptProcessorNode.onaudioprocess → WebSocket(audio_chunk)
→ SoulX-Duplug(ASR+VAD) → turn_state(speak) → VoiceSecretaryHandler._process_turn
→ VoiceSecretaryAgent.process_voice_command → TTS播报
```

**核心问题**：前端持续拾音 + 后端对所有 ASR 结果都做处理，导致：
- 用户随口说一声"嗯"，秘书就回复"我在听，请直接说出完整需求..."
- 用户和别人聊天，秘书也以为在跟它说话
- `should_ignore_utterance()` 被改为始终返回 `False`，所有语音都被当作有效输入

---

## 2. 目标交互模型

```
┌──────────────┐     点击唤醒      ┌──────────────┐     点击关闭/超时     ┌──────────────┐
│   待命态      │ ──────────────→ │   激活态      │ ──────────────────→ │   待命态      │
│  麦克风静默   │ ←────────────── │  正常拾音     │ ←────────────────── │  麦克风静默   │
│  不发音频     │     点击关闭     │  智能判断     │     30s无交互       │  不发音频     │
└──────────────┘                  └──────────────┘                      └──────────────┘
```

**关键原则**：
- **按钮是唯一唤醒入口**：点击麦克风按钮开启，再点击关闭，不需要唤醒词
- **激活态才拾音**：待命态下前端不发送 `audio_chunk`，后端不处理任何语音
- **智能意图门控**：激活态下，仅当用户说的是任务/查询指令才回复，闲聊/语气词静默

---

## 4. 详细实施方案

### 4.1 前端状态控制（核心改动）

**文件**：`console/src/features/core/voice/useVoiceSecretary.ts`

#### 4.1.1 新增 `active` 状态

在现有 `status` 之外新增独立的 `active` 开关：

```typescript
const [active, setActive] = useState(false);       // 是否处于激活态
const activeRef = useRef(false);                     // 同步引用，避免闭包问题
const autoSleepTimerRef = useRef<number | null>(null); // 自动休眠计时器
const AUTO_SLEEP_MS = 30_000;                        // 30秒无交互自动休眠
```

#### 4.1.2 修改音频发送逻辑

在 `processor.onaudioprocess` 回调中，增加 `active` 判断：

```typescript
processor.onaudioprocess = (event) => {
  const targetWs = wsRef.current;
  if (!targetWs || targetWs.readyState !== WebSocket.OPEN) return;
  // ★ 关键：未激活时不发送音频
  if (!activeRef.current) {
    pendingBufferRef.current = new Float32Array();
    return;
  }
  if (Date.now() < capturePausedUntilRef.current) {
    pendingBufferRef.current = new Float32Array();
    return;
  }
  // ... 原有音频块发送逻辑不变 ...
};
```

#### 4.1.3 新增 `activate()` / `deactivate()` 方法

```typescript
const activate = useCallback(() => {
  activeRef.current = true;
  setActive(true);
  setStatusText("语音秘书已激活，请说...");
  // 启动自动休眠计时器
  resetAutoSleepTimer();
}, []);

const deactivate = useCallback(() => {
  activeRef.current = false;
  setActive(false);
  clearAutoSleepTimer();
  // 清空待发送的音频缓冲
  pendingBufferRef.current = new Float32Array();
  setStatus("idle");
  setStatusText("语音秘书待命中（点击唤醒）");
}, []);

const resetAutoSleepTimer = useCallback(() => {
  if (autoSleepTimerRef.current) {
    window.clearTimeout(autoSleepTimerRef.current);
  }
  autoSleepTimerRef.current = window.setTimeout(() => {
    // 30秒无交互，自动休眠
    deactivate();
  }, AUTO_SLEEP_MS);
}, [deactivate]);

const clearAutoSleepTimer = useCallback(() => {
  if (autoSleepTimerRef.current) {
    window.clearTimeout(autoSleepTimerRef.current);
    autoSleepTimerRef.current = null;
  }
}, []);
```

#### 4.1.4 收到回复后重置休眠计时器

在 `handleServerMessage` 的 `assistant_result` 和 `assistant_ignored` 分支中：

```typescript
// 收到任何服务器响应 → 重置自动休眠计时器
if (activeRef.current) {
  resetAutoSleepTimer();
}
```

#### 4.1.5 修改 `start()` 方法

连接建立后默认进入**待命态**（不自动激活）：

```typescript
// ws.onopen 回调中：
setConnected(true);
setStatus("idle");
setStatusText("语音秘书已连接，点击麦克风唤醒");
// 不再自动设为 "已接入，直接说"
```

#### 4.1.6 暴露接口

```typescript
return {
  // ... 原有返回值 ...
  active,           // 新增：当前是否激活
  activate,         // 新增：激活
  deactivate,       // 新增：休眠
};
```

### 4.3 后端意图门控（智能判断是否需要回应）

**文件**：`src/copaw/agents/voice_secretary.py`

#### 4.3.1 修改 `should_ignore_utterance()` 

当前该方法始终返回 `False`，需要恢复过滤逻辑：

```python
def should_ignore_utterance(self, text: str) -> bool:
    normalized = str(text or "").strip()
    if not normalized:
        return True

    short_text = self._normalize_short_utterance(normalized)

    # 1. 纯语气词/短确认词 → 忽略，不回复
    if short_text in self.CASUAL_ACKS:
        return True

    # 2. 极短文本（≤2字）且不像任务请求 → 忽略
    if len(short_text) <= 2 and not self._looks_like_task_request(normalized):
        return True

    # 3. 闲聊/打招呼 但没有实际指令 → 忽略
    #    （之前会回复"你好，我在听..."，现在改为静默）
    if any(token in normalized for token in self.GREETING_PHRASES):
        if not self._looks_like_task_request(normalized):
            return True

    return False
```

#### 4.3.2 修改 `_maybe_handle_local_utterance()` 

去掉所有"我在听"/"请说出完整需求"之类的主动引导回复。只保留有实际信息量的本地处理：

```python
def _maybe_handle_local_utterance(self, text: str) -> VoiceSecretaryResult | None:
    normalized = str(text or "").strip()
    if not normalized:
        return None
    short_text = self._normalize_short_utterance(normalized)

    # 自我介绍查询 → 回复
    if any(token in normalized for token in self.SELF_QUERY_PHRASES):
        spoken = "我是你的语音秘书，可以帮你查信息、转办任务、提醒协同。"
        return self._build_local_result(
            title="语音秘书介绍",
            summary=spoken,
            spoken=spoken,
            text=normalized,
        )

    # 语音链路测试 → 短回复
    if any(token in normalized for token in self.HEALTH_CHECK_PHRASES):
        return self._build_local_result(
            title="语音连接正常",
            summary="语音链路正常",
            spoken="能听见。",
            text=normalized,
        )

    # 其他情况交给 _looks_like_task_request 判断
    return None
```

#### 4.3.3 修改 `process_voice_command()` 中的非任务指令处理

当前非任务指令会回复"我在听，请直接说出完整需求..."，改为**静默忽略**：

```python
async def process_voice_command(self, text, turn_context=None):
    normalized = str(text or "").strip()
    if not normalized:
        # 空文本 → 静默
        return VoiceSecretaryResult(spoken="", screen={})

    # 先检查是否应忽略
    if self.should_ignore_utterance(normalized):
        logger.info("VoiceSecretary ignored utterance user=%s text=%s", self.user_id, normalized[:60])
        # 返回空 spoken → 不触发 TTS，不播报
        return VoiceSecretaryResult(
            spoken="",
            screen={
                "kind": "voice_secretary_result",
                "title": "已收到语音",
                "summary": "当前语句不包含明确指令，静默处理。",
                "originalText": normalized,
            },
            route_result="ignored",
        )

    # 本地处理（自我介绍、链路测试等）
    local_result = self._maybe_handle_local_utterance(normalized)
    if local_result is not None:
        return local_result

    # 非任务指令 → 静默（不再说"我在听"）
    if not self._looks_like_task_request(normalized):
        logger.info("VoiceSecretary non-task, silent user=%s text=%s", self.user_id, normalized[:60])
        return VoiceSecretaryResult(
            spoken="",
            screen={
                "kind": "voice_secretary_result",
                "title": "非任务指令",
                "summary": "当前语句不像明确任务指令，已静默处理。",
                "originalText": normalized,
            },
            route_result="ignored",
        )

    # 任务指令 → 正常 IAP 转办（原有逻辑不变）
    # ...
```

## 6. 意图判断规则速查

| 用户说的 | 判断 | 行为 |
|----------|------|------|
| "嗯"、"啊"、"好的" | 语气词 | 静默，不回复 |
| "你好"、"喂" | 纯问候 | 静默，不回复 |
| "你是谁"、"你能做什么" | 自我查询 | 回复（短） |
| "听得见吗" | 链路测试 | 回复"能听见" |
| "帮我查一下XX" | 任务指令 | 正常处理+TTS |
| "提醒张总明天开会" | 任务指令 | IAP转办+TTS |
| "公司有什么制度" | SO查询 | IAP转办+TTS |
| "今天天气不错" | 闲聊 | 静默，不回复 |
| 空文本/噪声 | 无效 | 静默，不回复 |
