# CoPaw 1.5 改造计划

## 概述

基于 v1.4 的主动推送能力，1.5 聚焦**推送体验升级**与**回复转发**：将推送文案从「XX 对您说」改为 AI 管家口吻，并支持用户通过自然语言将回复转告给推送来源方。

---

## 一、改造目标

1. **推送文案 AI 改写**：不再使用「陈文豪 对您说：XXX」，改为管家式口吻，如「柏鑫，有一份来自陈文豪的新消息，他让你总结下今天的报告，需要我协助帮你处理吗？」
2. **回复转发**：当用户说「你告诉他 知道了」时，系统识别意图并自动将回复转告给推送来源方（陈文豪）。
3. **实现约定**：支持「他」指代推送来源；使用 LLM 做意图识别与文案改写；LLM 失败时使用降级文案。

---

## 二、核心功能设计

### 2.1 推送消息 LLM 改写

**改写前**：`陈文豪 对您说：让他总结下今天的工作包`

**改写后**：`陈文豪先生让我提醒您，需要总结一下今天的工作内容。请问您是否需要我协助整理？`

**实现要点**：
- 入队前调用 `rewrite_notification_message()` 进行 LLM 改写
- 支持 `【已改写】` 前缀，避免回复链路二次改写
- 入队格式：`{text, source_user_id, source_user_name}`

### 2.2 回复转发

**触发示例**：
- 「你告诉他 知道了」
- 「回复他好的」
- 「转告陈文豪收到了」

**流程**：
1. LLM 判断是否为「转发回复」意图
2. 解析目标用户名（支持「他」指代 session 的 push_source）
3. 改写回复内容为管家口吻
4. 写入 `event_bus.md`，格式：`- [ ] @通知Agent: 用户 @当前用户 请 @目标用户 【已改写】改写后的内容`

### 2.3 降级策略

- LLM 改写失败 → 使用原始文案或简单拼接
- 意图识别失败 → 不触发转发，走正常聊天流程

## 三、技术实现

### 3.1 新增文件

| 文件 | 作用 |
|------|------|
| `notification_llm.py` | `rewrite_notification_message`、`rewrite_reply_message`、`detect_reply_forward_intent` |

### 3.2 后端改动

| 文件 | 改动 |
|------|------|
| `notification_agent.py` | 入队前调用 `rewrite_notification_message`，支持 `【已改写】` 前缀 |
| `runner.py` | 回复转发：LLM 意图识别 → 改写 → 写入 event_bus；`from ..auth_db import get_users_by_name` |
| `custom_app.py` | chat 入口的回复转发逻辑，附带 `session_meta` |

### 3.3 前端改动

| 文件 | 改动 |
|------|------|
| `MainLayout`、`Header` | 兼容 string 和 dict 消息格式 |
| `sessionApi.createSession` | 支持 `pushSource`，写入 `meta.push_source_user_id/name` |
| `sessionApi.getSessionMeta()` | 供回复转发获取上下文 |
| `Chat/index.tsx` | 请求中附带 `session_meta` |

### 3.4 流式输出处理

**问题**：agentscope 流式解析后每个 chunk 为**累计全文**，非增量。

**修复**：`text = chunk_text` 覆盖，只保留最后一个 chunk，避免 `text += chunk_text` 导致重复输出。

## 五、实施优先级

**第一阶段（核心）**：
- [ ] 推送消息 LLM 改写 + 降级
- [ ] 回复转发：LLM 意图识别 + 改写 + event_bus
- [ ] Session 存储 source_user 元数据

**第二阶段（体验）**：
- [ ] 流式 chunk 重复问题修复
- [ ] 导入路径校验（`..auth_db` 而非 `.auth_db`）

**第三阶段（可选）**：
- [ ] 支持更多指代方式（如「回复那个人」）
- [ ] 推送文案风格可配置

## 六、测试验证点

**功能测试**：
- [ ] 推送消息显示为管家口吻
- [ ] 「你告诉他 知道了」正确转告给推送来源
- [ ] 「他」在推送会话中正确解析为来源用户
- [ ] LLM 失败时降级文案正常

**回归测试**：
- [ ] 推送无重复输出
- [ ] 回复转发无 `ModuleNotFoundError`
- [ ] 多用户推送、转发互不干扰

---

## 七、注意事项

- **user_id 一致性**：与 v1.4 相同，JWT、notification_agent、pull_messages 的 user_id 需一致
- **导入路径**：`runner.py` 中 `get_users_by_name` 需从 `..auth_db` 导入，否则会报 `copaw.app.runner.auth_db` 不存在
- **流式 chunk**：agentscope 返回累计全文，收集时只取最后一个 chunk
