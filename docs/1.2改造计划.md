### 1.2 改造计划：个性化欢迎语与智能档案引导系统

---

### 1. 改造目标

基于1.1版本的用户认证和档案系统，进一步优化用户体验：
- **智能欢迎语**：根据用户档案状态和部门信息，动态显示个性化欢迎语
- **情境化引导**：针对不同用户状态提供差异化的功能引导
- **温暖交互体验**：采用轻松友好的对话式交互方式

---

### 2. 核心功能设计

#### 2.1 智能欢迎语系统

**欢迎语逻辑：**
```
IF 用户首次登录（无档案）THEN
    显示："用户名你好，我今天能帮你做什么？我是埃弗赛科技的小智AI，可以帮助你解决问题。"
    称谓处理：总裁办用户显示为"某总"，其他部门显示姓名
    
ELSE IF 用户已有档案 THEN
    显示："欢迎回来，用户名，我今天能帮你做什么？"
    称谓处理：同样按部门区分称谓
```

**称谓规则：**
- 总裁办部门 → "张总"、"李总"（姓氏+总）
- 其他部门 → 正常称呼姓名

#### 2.2 档案状态检测机制

**检测方案A（文件内容分析）：**
```python
def check_user_profile_status(profile_id):
    """
    检测用户档案完成状态
    """
    profile_path = f"copaw_work/memory/employees/{profile_id}/MEMORY.md"
    
    # 读取档案内容
    try:
        with open(profile_path, 'r', encoding='utf-8') as f:
            content = f.read()
        
        # 判断是否为默认模板
        is_default = (
            "（在这里记录本员工的个人档案" in content or
            len(content.strip()) < 100 or
            content.count('#') < 3  # 标题数量不足
        )
        
        return not is_default  # True表示已有完整档案
        
    except FileNotFoundError:
        return False  # 档案文件不存在
```

### 4. 技术实现方案

#### 4.1 前端改造

**主要改动点：**
1. **聊天界面初始化逻辑**
   - 添加档案状态检测
   - 动态生成欢迎语
   - 根据状态显示不同引导按钮

2. **称谓处理模块**
   ```javascript
   function getUserGreeting(user) {
       const baseName = user.name;
       let displayName = baseName;
       
       // 总裁办特殊称谓
       if (user.department === "总裁办") {
           displayName = baseName.charAt(0) + "总";
       }
       
       // 根据档案状态调整欢迎语
       if (user.hasCompleteProfile) {
           return `欢迎回来，${displayName}，我今天能帮你做什么？`;
       } else {
           return `${displayName}你好，我今天能帮你做什么？我是埃弗赛科技的小智AI，可以帮助你解决问题。`;
       }
   }
   ```

3. **引导语动态显示**
   ```javascript
   function getGuideText(hasProfile) {
       if (!hasProfile) {
           return "立刻录入档案信息，让我们开启一段新的旅程吧！";
       } else {
           return "查看当前的工作进度";
       }
   }
   ```

#### 4.2 后端支持

**新增接口：**
```
GET /api/user/profile/status
- 功能：检测用户档案完成状态
- 返回：{ hasCompleteProfile: boolean, department: string }
```

**现有接口扩展：**
```
GET /api/auth/me
- 新增返回字段：department, profile_status
```

#### 4.3 档案检测逻辑

**前端实现：**
```javascript
async function checkProfileStatus() {
    try {
        const response = await fetch('/api/user/profile/status');
        const data = await response.json();
        return data.hasCompleteProfile;
    } catch (error) {
        console.error('检测档案状态失败:', error);
        return false;
    }
}
```
