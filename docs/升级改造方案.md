# 红智系统升级改造方案（党建场景）

> 依据文档：《红智系统赋能智慧党建场景探索报告_终极版》（2026-03-27）
> 结合现状：Aifscie Agent OS（CoPaw）当前前后端与双层 Agent 架构

## 1. 改造目标与原则

### 1.1 总体目标
将红智 OS 从“通用智能协作系统”升级为“党建场景化数字员工操作系统”，实现：

1. **基层减负**：把高频党务流程从“菜单+表单”转为“一句话代办”。
2. **组织关怀**：基于分身与记忆实现思想动态感知、主动提醒与温暖触达。
3. **活动协同**：把“群内通知催办”升级为“分身自动协同执行”。
4. **学习增效**：从刷题式学习升级为伴随式思政辅导与个性化学习闭环。
5. **评优评先**：建立党员测评与支部评比闭环，支撑优秀党员/优秀支部评选。
6. **时效响应**：围绕党中央最新指示精神形成“日级更新、周级复盘、月级专题分析”机制。

### 1.2 建设原则
- 政治合规优先（内容、流程、权限、审计全链路可控）
- 业务闭环优先（发起-执行-留痕-复盘）
- 低负担交互优先（秘书会话入口统一）
- 渐进式演进（先场景试点，再组织级推广）

## 2. 当前系统能力基线（可直接复用）

### 2.1 架构基线
- 后端：FastAPI + AgentScope + SQLite 持久化
- 前端：React + 员工端 `/app` + 管理端 `/admin`
- Agent：`SO`（系统编排）+ `PIA`（员工分身）双层协作
- 协议：IAP（Agent 间通信封装）

### 2.2 已具备能力
1. **红智秘书统一入口**：已在员工端形成秘书对话入口与会话恢复链路。
2. **消息分发链路**：`event_bus -> notification_agent -> message_store -> /api/messages/pull` 已成闭环。
3. **记忆沉淀机制**：公共记忆与员工私有记忆目录分层，具备长期沉淀基础。
4. **会话管理能力**：已有会话中心、触发时间展示、当天筛选等界面基础。

### 2.3 与党建场景的主要差距
1. 缺少“党建专属技能包”（三会一课、党费、组织生活、民主评议等）。
2. 缺少“组织思想温度”指标与风险预警模型。
3. 缺少“活动组织自动化执行链”（报名、提醒、确认、统计）。
4. 缺少“思政学习个性化推荐 + 学习效果评估”闭环。
5. 缺少“党员测评（优秀/合格/一般）与优秀党员评选”标准化机制。
6. 缺少“支部评比（含选人提拔支撑、优秀支部评选）”量化模型与流程引擎。
7. 缺少“党中央指示精神时效跟踪 + 企业特征化分析 + 自动成文”能力。

## 3. 目标能力蓝图（对应报告四大探索）

## 3.1 场景A：秘书代办极简党务
**目标**：一句话发起党务流程并自动执行。

### 改造点
- 新增党建流程技能：
  - 会议发起（时间/地点/参会对象）
  - 通知生成与分发
  - 备案审批流转
  - 纪要模板自动生成
- 秘书前端新增“党务任务卡”组件（状态：待确认/执行中/已完成）
- 关键步骤引入“二次确认”机制（防误触）

### 产出
- “一句话发起三会一课”可端到端完成
- 形成标准化流程留痕（审批、通知、纪要）

## 3.2 场景B：组织关怀与思想动态感知
**目标**：从被动问卷转为主动、持续、低打扰的组织关怀。

### 改造点
- 新增“思想动态画像”服务：
  - 输入：学习行为、互动语义、任务状态、公开表达
  - 输出：活跃度、压力趋势、学习专注度、关怀建议
- 支持领导视角“分身摘要对话”（不直接暴露敏感原文）
- 关怀触发器：政治生日、长期高负荷、学习中断等

### 产出
- 组织关怀看板（趋势图 + 建议动作）
- 关怀消息可通过分身自动下发并追踪反馈

## 3.3 场景C：党建活动分身协同
**目标**：活动组织从人工催办升级为分身自动协同。

### 改造点
- 活动中心模板化：报名、投票、签到、反馈统一模板
- 与消息链路打通：活动事件直接投递到目标 PIA 信箱
- 授权机制：在授权范围内，PIA 可代为完成“意向确认/回执”
- 结果回流：活动执行状态自动更新到会话与管理看板

### 产出
- 活动组织时效显著提升
- 人工催办频次与漏回执率下降

## 3.4 场景D：伴随式思政辅导员
**目标**：把学习从“任务打卡”升级为“工作伴随式学习”。

### 改造点
- 建立党建知识库（政策、党史、案例、组织制度）
- 在秘书对话和写作场景中进行上下文推荐（理论金句、政策依据）
- 个性化学习路径：按岗位/薄弱点推送微课
- 学习评估：从“时长”转向“理解-应用-反馈”三维评估

### 产出
- 学习推荐精准化
- 学习成果可量化并可追溯

## 3.5 场景E：党员测评与优秀党员评选
**目标**：建立公平、可解释、可追溯的党员测评体系。

### 改造点
- 测评等级标准化：`优秀`、`合格`、`一般`。
- 测评维度建议：政治表现、组织参与、学习成效、协同贡献、群众反馈。
- 测评流程：自评 -> 互评/支委评 -> 组织复核 -> 公示归档。
- 系统支持自动汇总评分与排序，形成“优秀党员候选清单”。

### 产出
- 党员测评台账与排名看板
- 优秀党员评选报告（含评分依据与过程留痕）

## 3.6 场景F：支部评比与选人提拔支撑
**目标**：形成支部横向对比能力，支撑优秀支部评选与干部选拔参考。

### 改造点
- 支部评比分层指标：组织建设、活动质量、学习覆盖、执行效率、先锋案例。
- 当支部数量 >= 3 时，自动触发“优秀支部”评选流程与榜单生成。
- 融入选人提拔支撑：按岗位画像输出“可培养对象/重点关注对象”建议。
- 提供“证据链下钻”：从总分可追溯到具体事件、会话与任务记录。

### 产出
- 支部评比月报/季报
- 优秀支部榜单与提拔建议辅助报告

## 3.7 场景G：党中央指示精神时效响应与企业分析成文
**目标**：突出产品时效性，把党中央最新指示快速转化为企业可执行方案。

### 改造点
- 建立“指示精神雷达”：政策更新自动抓取、去重、分级、时效告警。
- 建立“企业特征画像”：行业、规模、组织架构、痛点、年度重点任务。
- 建立“匹配分析引擎”：将指示精神与企业画像进行差距分析与行动建议生成。
- 自动生成文档：
  - 《指示精神解读简报（日/周）》
  - 《本企业贯彻落实分析报告（月）》
  - 《专项整改与提升方案（专题）》

### 产出
- 指示精神响应时效看板（T+1/T+3达标率）
- 企业定制分析文档自动化产出能力

## 4. 分层技术改造方案

## 4.1 前端（员工端）
1. 秘书页增加党建模式入口（默认保持统一会话体验）。
2. 新增模块：
   - 党务任务卡
   - 活动协同面板
   - 关怀提醒面板
   - 学习推荐卡
   - 党员测评面板（优秀/合格/一般）
   - 支部评比面板（当支部>=3时支持优秀支部评选）
   - 指示精神响应中心（时效看板 + 分析报告入口）
3. 会话中心增加党建标签筛选（党务/活动/学习/关怀/测评/评比/指示）。

## 4.2 后端（服务与路由）
1. 新增 `party_affairs_service`：流程编排、模板渲染、状态流转。
2. 新增 `organization_care_service`：动态评估、关怀策略、提醒计划。
3. 新增 `learning_coach_service`：知识检索、推荐生成、学习评估。
4. 新增 `member_evaluation_service`：党员测评规则、评分计算、候选推荐、公示归档。
5. 新增 `branch_ranking_service`：支部评比、优秀支部评选、选人提拔辅助分析。
6. 新增 `directive_intelligence_service`：党中央指示解析、时效跟踪、企业特征匹配、自动成文。
7. 扩展现有消息与会话接口：支持党建业务元数据（场景、任务ID、状态）。

## 4.3 Agent 与技能层
1. 新建党建技能包：
   - `schedule_party_meeting`
   - `generate_party_notice`
   - `dispatch_activity_task`
   - `learning_recommendation`
   - `care_reminder`
   - `member_evaluation_scoring`
   - `excellent_member_selection`
   - `branch_ranking`
   - `excellent_branch_selection`
   - `directive_digest_and_enterprise_analysis`
2. SO 增加党建任务路由策略（按组织、角色、优先级分发）。
3. PIA 增加可授权动作白名单（代确认、代提醒、代统计）。

## 4.4 数据与记忆层
1. 新增党建业务表（建议）：
   - `party_events`
   - `party_event_participants`
   - `care_signals`
   - `learning_profiles`
   - `member_evaluations`
   - `branch_rankings`
   - `directive_updates`
   - `enterprise_analysis_reports`
2. 记忆增加场景标签与生命周期管理（防止无效膨胀）。
3. 审计增强：所有“自动代办动作”必须记录触发来源、时间、责任人。

## 4.5 安全与合规
1. 政治内容安全策略：敏感语义识别 + 审核开关。
2. 权限控制：按组织层级、岗位角色精细化授权。
3. 数据治理：分级脱敏、最小可见范围、全链路审计追踪。
4. 评比公平性：评分规则版本化、评分过程可回放、结果可申诉。
5. 指示溯源：党中央指示来源、发布时间、解读版本必须可追踪。

## 5. 分阶段实施路线（12周）

### 阶段1（第1-3周）：MVP 场景打通
- 打通“一句话发起三会一课”全链路
- 上线活动通知/回执最小闭环
- 上线党员测评最小版本（优秀/合格/一般）
- 交付可演示 Demo（面向客户验证）

### 阶段2（第4-7周）：协同、关怀与评比增强
- 上线活动协同面板与自动提醒
- 上线思想动态基础画像与关怀策略
- 上线支部评比与优秀支部评选（支部>=3自动触发）
- 形成组织级看板（试点部门）

### 阶段3（第8-10周）：学习辅导与指示响应闭环
- 接入党建知识库与推荐引擎
- 上线伴随式思政辅导
- 建立学习效果评估报表
- 上线党中央指示精神时效响应与企业分析成文能力

### 阶段4（第11-12周）：治理与规模化
- 完成权限、审计、脱敏与运维策略
- 完成评比公平性与指示溯源机制固化
- 形成标准化实施包（模板+配置+验收清单）
- 进入多组织复制推广

## 6. 验收指标（建议）

### 6.1 效率指标
- 党务流程平均处理时长下降 ≥ 40%
- 活动回执完成率提升 ≥ 30%
- 人工催办次数下降 ≥ 50%

### 6.2 体验指标
- 秘书入口任务一次完成率 ≥ 80%
- 用户满意度（基层党务）≥ 4.5/5
- 日活跃使用率与次日留存持续增长

### 6.3 组织价值指标
- 学习完成质量（非时长）提升
- 组织关怀触达率提升
- 关键风险事件前置预警命中率提升
- 优秀党员评选准确性与认可度持续提升
- 优秀支部评选公信力（申诉率下降）持续提升

### 6.4 时效与指示落实指标
- 党中央新指示 T+1 摘要生成达标率 ≥ 95%
- 企业定制落实分析报告 T+3 交付达标率 ≥ 90%
- 指示到行动项自动拆解准确率 ≥ 85%

## 7. 风险与应对

1. **风险：自动化动作越权**
   - 应对：授权白名单 + 高风险动作二次确认 + 审计追责。

2. **风险：内容合规与政治表达偏差**
   - 应对：党建语料对齐、策略拦截、人工复核通道。

3. **风险：基层接受度不足**
   - 应对：先试点后推广，保留人工兜底流程，强化培训与反馈闭环。

4. **风险：数据质量不足影响推荐效果**
   - 应对：数据标准化、标签治理、周期性评估与回归优化。

5. **风险：测评/评比结果争议**
   - 应对：规则公开、证据链可追溯、复议流程与人工仲裁通道。

6. **风险：指示精神理解偏差导致执行偏航**
   - 应对：权威来源比对、解读版本管理、关键报告人工复核。

## 9. 企业特征化文档生成规范（新增）

### 9.1 企业特征输入模板
- 企业类型（央国企/民企/高校/街道）
- 组织规模（支部数量、党员人数、层级结构）
- 行业属性（制造、能源、教育、政务等）
- 当前痛点（基层负担、协同低效、学习形式化等）
- 年度重点任务（党建品牌、专项整治、人才提拔）

### 9.2 自动生成文档清单
1. 《党中央指示精神与本企业落实差距分析报告》
2. 《本企业党员测评与优秀党员评选实施细则》
3. 《本企业支部评比与优秀支部评选办法（含选人提拔支撑）》
4. 《季度党建重点任务推进报告（含时效预警）》

### 9.3 成文机制
- 日更：指示精神摘要与影响提示
- 周更：落实进展、偏差项与纠偏建议
- 月更：企业专题分析与制度优化建议

## 10. 结论

基于现有红智 OS 的双层 Agent、秘书入口、消息分发和记忆体系，党建场景升级不需要推倒重来，重点是**“场景技能化 + 协同自动化 + 治理可控化 + 时效响应化 + 评优标准化”**。建议以“秘书代办 + 活动协同 + 党员测评”作为首期抓手，快速形成可见价值，再迭代支部评比、组织关怀与学习辅导能力，完成从“功能平台”到“组织智能体系统”的跃迁。
