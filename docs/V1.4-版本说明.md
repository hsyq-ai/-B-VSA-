# CoPaw v1.4 版本说明

## 概述

v1.4 主要新增**主动推送**能力：当用户被他人@通知时，系统会自动创建新会话并跳转，实现跨用户的消息触达。

---

## 一、核心功能

### 1. 主动推送链路

- **触发**：`event_bus.md` 新增 `@通知Agent` 任务
- **监听**：`event_listener` 监听文件变更，分发到 `notification_agent`
- **解析**：`notification_agent` 解析目标用户，将消息推入 `message_queue`
- **轮询**：前端在 `MainLayout` 中每 3 秒轮询 `GET /api/messages/pull`
- **展示**：收到消息后自动 `createSession` + `navigate`，打开新聊天框

### 2. 聊天指令触发

用户在聊天框输入以下格式时，会自动写入 `event_bus` 并触发推送：

| 格式示例 | 说明 |
|---------|------|
| `通知贺柏鑫 让他把今天的工作内容总结下` | 通知 + 用户名 + 消息 |
| `请@贺柏鑫 整理文档` | 请 + @用户名 + 消息 |
| `让陈文豪 开会` | 让 + 用户名 + 消息 |

**正则**：`(?:请|让|通知)\s*@?\s*(\w+)\s+(.+)`

目标用户名需在 `auth.db` 的 `users` 表中存在。

### 3. 姓名从数据库查询

- 移除硬编码用户名（如 `"陈文豪"`）
- 新增 `get_user_name_by_id_or_profile_id(val)`：按 `users.id` 或 `users.profile_id` 查询姓名
- `runner` 和 `custom_app` 均通过该函数获取当前用户姓名，用于通知任务中的来源显示

## 二、技术改动

### 后端

| 文件 | 改动 |
|------|------|
| `_app.py` | `MESSAGE_QUEUE`、`pull_messages`、`test-push`、`debug` 接口；启动时注册 `event_listener` |
| `event_listener.py` | 监听 `event_bus.md`，分发 `@通知Agent` 任务 |
| `notification_agent.py` | 解析 `请 @目标 消息`，调用 `get_users_by_name`，推入 `message_queue` |
| `auth_db.py` | 新增 `get_user_name_by_id_or_profile_id` |
| `runner.py` | 指令检测正则、写入 `event_bus`、来源姓名查询 |
| `custom_app.py` | chat 拦截、指令处理、来源姓名查询 |

### 前端

| 文件 | 改动 |
|------|------|
| `MainLayout/index.tsx` | 轮询逻辑从 Chat 移至此处，登录后任意页面均可收到推送 |
| `Chat/sessionApi/index.ts` | `createSession` 支持 `pushMessage`；`getSession` 本地有 `messages` 时直接返回 |
| `Header.tsx` | 新增「测试推送」按钮，用于手动验证推送链路 |
| `Chat/index.tsx` | `ChatWithKey` 用 `key={chatId}` 强制 remount，侧边栏会话列表刷新 |

### 消息格式

为兼容 Chat 库，为推送消息补充 `type: "message"`、`status: "completed"` 等字段。

## 三、调试与测试

### 调试接口

- `GET /api/messages/debug`（需登录）：返回 `user_id`、`pending_count`，用于验证链路
- `POST /api/messages/test-push`（需登录）：向当前用户队列注入测试消息

### 测试方式

1. **Header「测试推送」按钮**：点击后注入并拉取，自动创建新会话并跳转
2. **聊天指令**：用户 A 输入 `请@用户B 消息`，用户 B 登录后约 3 秒内收到推送
3. **手动写入 event_bus**：在 `copaw_work/event_bus.md` 追加 `- [ ] @通知Agent: 用户 @来源 请 @目标 消息`

## 四、关键文件

| 文件 | 作用 |
|------|------|
| `copaw_work/event_bus.md` | 任务格式：`- [ ] @通知Agent: 用户 @来源 请 @目标 消息` |
| `event_listener.py` | 监听 event_bus，分发到 Agent |
| `notification_agent.py` | 解析目标用户，推入 message_queue |
| `_app.py` | MESSAGE_QUEUE、pull_messages、test-push |
| `MainLayout/index.tsx` | 轮询、创建会话、跳转 |
| `sessionApi/index.ts` | createSession(pushMessage)、getSession 逻辑 |
| `runner.py` / `custom_app.py` | 指令检测、写入 event_bus |
| `auth_db.py` | get_user_name_by_id_or_profile_id、get_users_by_name |

---

## 五、注意事项

- **user_id 一致性**：JWT 的 `user_id`、`notification_agent` 的 `target_user_id`、`pull_messages` 的 `user_id` 均对应 `users.id`，三者需一致
- **构建产物**：`npm run build` 输出到 `CoPaw/src/copaw/console/`，无需手动复制
- **轮询范围**：登录后任意页面都会轮询，不限于 `/chat`
