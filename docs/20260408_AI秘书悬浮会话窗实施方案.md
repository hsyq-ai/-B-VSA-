# AI 秘书悬浮会话窗实施方案

## 目标

将现有 `FloatingAiDock` 从“说明型 AI 面板”改造为“会话主导的 AI 秘书窗口”，满足以下体验目标：

1. 打开即进入可互动会话。
2. 顶部提供系统跳转入口。
3. 能接收被动消息通知并展示最新摘要。
4. 可一键把消息交给秘书处理。
5. 保留当前页上下文承接能力。

## 实施范围

### 1. 结构改造
- 将聊天区提升为弹窗主体。
- 将原大卡片式建议区改为紧凑快捷动作栏。
- 增加“消息与上下文”折叠区。

### 2. 通知联动
- 监听 `copaw-push-session-updated`。
- 复用 `sessionApi.getSessionList()` 中的推送会话数据。
- 在悬浮按钮和弹窗顶栏展示未读/新到消息提示。

### 3. 快捷交互
- 提供固定动作：`处理最新消息`、`总结待办`、`生成今日安排`。
- 支持消息卡片上的 `让我处理` / `去查看`。
- 保留当前场景动作与页面上下文注入。

## 交互结构

1. 顶栏：在线状态、主会场/通知/工作台/收起。
2. 快捷动作：秘书处理入口与场景动作。
3. 主体：内嵌会话 iframe。
4. 底部：快捷输入与常用 prompt。
5. 折叠区：最新 3 条消息、更多入口、当前页上下文。

## 数据复用方案

### 会话能力
继续复用现有秘书会话：
- `createSecretarySessionStorage`
- `sessionApi.createSession()`
- `workspace-embed` iframe
- `copaw-hidden-submit`

### 通知能力
继续复用现有推送链路：
- `usePushBridge` 轮询 `/messages/pull`
- `copaw-push-session-updated` 事件
- `sessionApi.getSessionList()` 中的推送会话元数据
