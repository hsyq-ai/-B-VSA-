# 科研部仪表盘菜单改造指南

## 1. 目标

科研部用户登录后，仪表盘展示两个一级菜单：

- 科研规划
- 科研工具

并基于二级菜单触发对应的提示词与 Skill。

## 2. 当前系统改造入口

前端：

- `console/src/components/employee/EmployeeSidebar.tsx`  
  负责左侧菜单渲染、场景触发、科研部菜单分支逻辑。
- `console/src/pages/DashboardSkillTemplates/index.tsx`  
  负责管理员配置“按部门展示的仪表盘项”。
- `console/src/api/modules/dashboardSkills.ts`  
  负责 `/dashboard-skills` 与 `/dashboard-skills/resolve` 调用。

后端：

- `src/copaw/app/prompt_templates_store.py`  
  内置模板定义（`trigger_key`、`display_name`、`prompt_text`、`skill`、`category`）。
- `src/copaw/app/dashboard_skill_rules_store.py`  
  部门到 trigger 列表的规则存储与解析。
- `src/copaw/app/routers/dashboard_skills.py`  
  仪表盘规则管理与解析 API。

## 3. 菜单与触发映射（科研部）

建议将科研部仪表盘 trigger 全部放到 `category = "dashboard"`，并按以下命名：

| 一级菜单 | 二级菜单 | trigger_key（建议） | skill（建议） | prompt_text（建议方向） |
|---|---|---|---|---|
| 科研规划 | 选题研判 | `dashboard-research-topic` | `file_reader`（过渡）/ `research_topic`（新建） | 科研问题甄选、项目卡点排障、主动询问澄清 |
| 科研规划 | 质量评估 | `dashboard-research-quality` | `file_reader`（过渡）/ `research_quality`（新建） | 论文质量评估、实验设计、质量批判性分析 |
| 科研规划 | 头脑风暴 | `dashboard-research-brainstorm` | `file_reader`（过渡）/ `research_brainstorm`（新建） | 科研灵感启迪、多方向方案发散 |
| 科研工具 | 知识检索 | `dashboard-research-search` | `file_reader`（过渡）/ `research_search`（新建） | 查文献、查论文，输出检索路径和结论 |
| 科研工具 | 数据分析 | `dashboard-research-data` | `file_reader`（过渡）/ `research_data`（新建） | 上传图片/文档/其他文件后进行结构化分析 |
| 科研工具 | 科研创作 | `dashboard-research-writing` | `docx`（Word） + `pdf`（导出） | 生成科研写作草稿、结构化章节与可导出文档 |
| 科研工具 | 论文生成 | `dashboard-research-paper-gen` | `file_reader` + `docx` + `pdf`（编排） | 查阅相关论文资料并生成论文草稿 |
| 科研工具 | 业界跟踪 | `dashboard-research-tracking` | `news` | 最新业界科研进展实时跟进与摘要 |

说明：

- 现有可直接使用 skill：`file_reader`、`docx`、`pdf`、`news`。
- 如果希望“括号内容=专属触发能力”，建议逐步新增 8 个科研专属 skill，再把上表中的“过渡 skill”替换掉。

## 4. 实施步骤

### Step A：新增/更新内置模板（后端）

在 `src/copaw/app/prompt_templates_store.py` 的 `_DEFAULT_TEMPLATES` 中新增 8 条科研模板，要求：

- `category: "dashboard"`
- `template_type: "scene"`
- `trigger_key` 使用上表建议值
- `display_name` 与菜单文字一致
- `prompt_text` 写成“先澄清再执行”的科研助手提示词
- `skill` 先指向可用 skill（如 `file_reader`/`news`/`docx`/`pdf`）

### Step B：更新科研部默认规则（后端）

在 `src/copaw/app/dashboard_skill_rules_store.py` 的 `_DEFAULT_RULES["departments"]["科研部"]` 改为 8 个科研 trigger_key 列表。

建议保持 `default` 不变，避免影响非科研部门。

### Step C：改造科研部仪表盘菜单结构（前端）

在 `console/src/components/employee/EmployeeSidebar.tsx`：

- 新增科研部专用菜单常量，按两个一级菜单组织：
  - 科研规划：3 个子项
  - 科研工具：5 个子项
- 在 `isResearchDept` 分支中，将原 `dashboard-group` 替换为“科研规划 + 科研工具”两个一级菜单。
- 二级菜单 `key` 推荐直接使用对应 `trigger_key`，这样可复用现有 `sceneConfig[nextKey]` 触发逻辑。
- 若保留 `dashboard-group`（兼容旧页面），也可用双层 children 实现，但不如拆成两个一级菜单直观。

### Step D：场景配置绑定（前端）

在 `BASE_SCENES` 增加 8 个科研项，保证每个菜单 key 能命中：

- `label`
- `triggerKey`
- `sessionName`
- `skill`
- `templateType: "scene"`

### Step E：管理员配置页对齐（前端）

`console/src/pages/DashboardSkillTemplates/index.tsx` 中：

- 加载规则后，将科研部默认值替换为新 8 个 trigger。
- 校验 `templateOptions` 可见这 8 个项，便于管理员后续调整。

## 5. 最小可用改法（推荐）

如果希望先快速上线：

- 不新建科研专属 skill。
- 先用已有 skill 做映射：
  - 规划类（选题研判/质量评估/头脑风暴）→ `file_reader`
  - 检索/数据分析/论文生成 → `file_reader`
  - 科研创作 → `docx`（必要时补 `pdf`导出）
  - 业界跟踪 → `news`
- 后续迭代时再细分专属 skill，触发键不变，前端无需再次改菜单。

## 6. 验收清单

- 科研部账号登录后，左侧出现且仅出现目标一级菜单：科研规划、科研工具（按产品期望）。
- 每个二级菜单点击后可创建会话，并写入正确 `triggerKey`。
- `/dashboard-skills/resolve?department=科研部` 返回 8 个目标 trigger。
- 管理员页面可查看并勾选这 8 个科研 trigger。
- 非科研部账号菜单行为不回归。

## 7. 回滚方案

- 前端回滚：恢复 `EmployeeSidebar.tsx` 中科研部分支为旧 `dashboard-group` 结构。
- 后端回滚：恢复 `prompt_templates_store.py` 与 `dashboard_skill_rules_store.py` 的原始科研触发列表。
- 数据回滚：将仪表盘规则文件恢复到变更前版本（若已持久化到运行目录）。
