### 2️⃣ **MemoryManager 并发安全隔离**（阶段一）

#### **改动文件**
- `/CoPaw/src/copaw/app/runner/runner.py`
- `/CoPaw/src/copaw/agents/memory/memory_manager.py`

#### **核心改进**
- ✅ 添加异步锁 `_memory_lock` 保护 profile_id 设置
- ✅ 每个请求独占访问 memory_manager，防止状态污染
- ✅ 彻底解决多用户同时聊天时的档案串场问题

#### **技术实现**
```python
class AgentRunner(Runner):
    def __init__(self):
        super().__init__()
        self._memory_lock = asyncio.Lock()  # ← 并发控制锁

async def handle_request(self, request, msgs):
    if self.memory_manager is not None:
        async with self._memory_lock:
            # 设置当前请求的 profile_id
            self.memory_manager.set_current_profile_id(user_id)
            
            # 创建 agent（此时 profile_id 已正确设置）
            agent = CoPawAgent(
                memory_manager=self.memory_manager,
                ...
            )
```

### 3️⃣ **读写锁优化 ChatManager**（紧急修复）

#### **改动文件**
- `/CoPaw/src/copaw/app/runner/manager.py`

#### **核心改进**
- ✅ 替换单一锁为读写锁（ReadWriteLock）
- ✅ 读操作共享（多人可同时查询）
- ✅ 写操作排他（创建/更新时阻塞）
- ✅ 解决创建档案时阻塞注册和管理员页面的问题

#### **技术实现**
```python
class ChatManager:
    def __init__(self):
        # 读写锁机制
        self._read_lock = asyncio.Lock()      # 读锁（共享）
        self._write_lock = asyncio.Lock()     # 写锁（排他）
        self._readers_count = 0               # 当前读者数量
    
    async def list_chats(self):
        await self._acquire_read()   # 获取读锁（共享）
        try:
            return await self._repo.filter_chats(...)
        finally:
            await self._release_read()  # 释放读锁
    
    async def create_chat(self, spec):
        await self._acquire_write()  # 获取写锁（排他）
        try:
            await self._repo.upsert_chat(spec)
            return spec
        finally:
            await self._release_write()
```

## 🔍 问题修复记录

### Bug 1: logger 未定义导致登录失败
- **现象**: Internal Server Error，所有用户无法登录
- **原因**: 添加日志时忘记导入 logger
- **修复**: 添加 `import logging` 和 `logger = logging.getLogger(__name__)`

### Bug 2: 并发导致 profile_id 串场
- **现象**: 黄丫丫问"我是谁"，AI 回答冯程程的信息
- **原因**: 多用户同时设置 memory_manager 的 profile_id，互相覆盖
- **修复**: 添加 `asyncio.Lock()` 保护设置过程

### Bug 3: 创建档案阻塞其他操作
- **现象**: 用户在创建档案时，其他人无法注册，管理员页面卡住
- **原因**: 单一排他锁导致所有操作串行
- **修复**: 引入读写锁，读操作共享，写操作排他

## ⚠️ 待优化问题

### 1️⃣ **长期架构优化**（阶段二）
**问题**: 单例 AgentRunner 共享，理论上仍有竞态条件风险

**现状**:
- 使用 asyncio.Lock() 临时解决
- 适合 10-20 人并发
- 如果并发增长到 50+，需要重构

**建议方案**:
```python
class SessionPoolManager:
    """为每个用户创建独立的 Runner 实例"""
    
    def __init__(self):
        self.runners = {}  # user_id -> AgentRunner
        self.last_used = {}
    
    async def get_runner(self, user_id: str):
        if user_id not in self.runners:
            runner = AgentRunner()
            await runner.initialize()
            self.runners[user_id] = runner
        return self.runners[user_id]
```

**优点**:
- ✅ 完全物理隔离
- ✅ 支持 50+ 并发
- ✅ 零串场风险

**缺点**:
- ⚠️ 内存开销增加（每用户~500MB）
- ⚠️ 需要 1-2 天开发时间

### 3️⃣ **Session 文件存储优化**
**问题**: JSONSession 文件随用户增长而膨胀

**现状**:
- 每个会话一个 JSON 文件：`{profile_id}_{session_id}.json`
- 10 个用户 × 平均 5 个会话 = 50 个文件
- 长期运行可能达到数千个文件

**建议方案**:
- 引入 SQLite 存储会话历史
- 定期归档旧会话
- 添加会话清理策略（如 30 天未访问自动删除）

**优先级**: 低（当前 10 人无压力）
