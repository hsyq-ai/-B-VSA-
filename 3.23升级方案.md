# 3.23 升级方案（实施路线图）

> 目标：在不影响现有“联络站（真实员工）”定位的前提下，引入“数字员工（多 Agent）”能力，并完成技能模板化、SSE 重连与会话持久化、定时任务中心与技能安全治理。

---

## 一、总体目标与原则

- **双轨协作**：联络站 = 真实员工；数字员工 = AI Agent。
- **可演进**：提示词模板升级为 Skill 模板，支持导入与版本化。
- **稳定优先**：SSE 重连与会话持久化先于功能扩展落地。
- **安全兜底**：技能执行需安全扫描与运行隔离。

---

## 二、产品结构调整

### 1. 左侧导航结构
- **联络站**（真实员工协作）
  - 发起：协同任务 / 活动事项 / 通知会议 / 投票
  - 选定：部门 → 员工 → 触发协作通知
- **数字员工**（AI 协作）
  - Agent 切换器（多角色、多能力）
  - 每个 Agent 独立提示词 / Skill / 记忆档案

### 2. 控制台一致性
- 统一会话入口体验，避免用户混淆“人类协作”与“AI 协作”。

---

## 三、实施路线（P0 / P1 / P2）

### **P0：关键稳定性 + 基础多 Agent**
目标：先解决稳定性与用户体验，再引入数字员工入口。

1. **数字员工入口与 Agent 切换器**
   - 新增“数字员工”菜单
   - Agent 列表可切换会话
   - 每个 Agent 有独立提示词入口

2. **SSE 重连 + 会话持久化策略**
   - SSE 断线重连（前端）
   - 会话消息增量持久化（后端）
   - 解决“会话偶发丢失/失效”问题

3. **会话上下文隔离**
   - 多 Agent 的会话与记忆隔离
   - 保证切换不会交叉污染

**交付验收**
- 任何 Agent 切换不丢消息
- 断线后可恢复会话流
- 会话列表稳定可回溯

---

### **P1：Skill 模板体系 + 定时任务中心**
目标：把提示词模板升级为可复用技能与任务编排。

1. **Skill 模板体系**
   - 将现有“提示词模板”升级为 Skill 模板
   - 支持：
     - 分类（通知 / 总结 / 调研 / 生成 / 审核）
     - 绑定 Agent
     - 导入（Zip/目录）
     - 版本化（可追溯修改）

2. **定时任务中心**
   - 统一调度管理
   - 任务配置项：
     - 执行时间、频率
     - 触发 Agent
     - 输出渠道（会话/通知）
   - 典型场景：
     - 早 8 点科技新闻汇总

**交付验收**
- Skill 模板在后台可维护、可导入
- 定时任务可配置并按时执行
- 任务结果可回填到会话

---

### **P2：安全扫描 + 运行隔离**
目标：降低技能执行风险，提升企业级可控性。

1. **技能安全扫描**
   - 检测危险命令与高风险行为
   - 拦截或要求二次确认

2. **运行环境隔离**
   - 文件生成 / 执行类技能与主进程隔离
   - 沙箱化（限制写路径、限制命令）

**交付验收**
- 危险指令不会直接执行
- 任务执行环境与主系统隔离

---

## 四、关键技术改造点

- **会话层**
  - SSE 断线重连
  - 会话持久化增强
  - 多 Agent 会话隔离

- **Skill 层**
  - Skill 模板化
  - Skill 导入机制
  - Skill 版本追踪

- **任务层**
  - 定时任务中心
  - 任务执行结果回填

- **安全层**
  - 风险扫描
  - 环境隔离

---

## 五、风险与对策

- **风险：多 Agent 会话混乱**
  - 对策：明确 sessionId + agentId 绑定
- **风险：技能执行安全**
  - 对策：危险命令检测 + 沙箱执行
- **风险：任务调度失败**
  - 对策：执行日志 + 失败重试 + 结果通知

---

## 六、交付节奏建议

1. **P0（1-2 周）**
   - Agent 入口 + SSE 重连 + 会话隔离
2. **P1（2-3 周）**
   - Skill 模板 + 定时任务中心
3. **P2（2-4 周）**
   - 安全扫描 + 运行隔离

---

## 七、最终形态

- 联络站 = 真实员工协作入口  
- 数字员工 = AI Agent 协作入口  
- Skill = 可复用能力模块  
- 定时任务 = 自动化执行中心  
- 安全与隔离 = 企业级合规与稳定保障
