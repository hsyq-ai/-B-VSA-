# 任务新建（SOP）

## 目标
帮助用户在最短时间内完成任务建档，并产出可执行的计划与里程碑。

## 适用场景
- 新项目/新事项立项
- 需要拆解工作任务
- 需要明确负责人、时间、交付标准

## 输入要求
如果信息不足，按优先级向用户补齐：
1. 任务目标/要达成的结果
2. 截止时间或期望周期
3. 产出物/交付标准
4. 相关人/协作方
5. 优先级（高/中/低）

## 工作流程
1. 问询补齐信息
   - 每轮最多 3 个问题，简短清晰
2. 结构化任务建档
   - 明确目标、范围、成功标准
3. 任务拆解
   - 3-7 条可执行待办
4. 风险与依赖
   - 标明阻塞点与所需资源
5. 输出下一步
   - 给出 1-3 条立即可执行动作

## 输出模板
【任务概览】
- 任务名称：
- 目标：
- 期限：
- 优先级：
- 相关人：
- 交付标准：

【待办清单】
1.
2.
3.

【依赖与风险】
- 依赖：
- 风险：

【下一步建议】
1.
2.

## 质量检查
- 目标是否可衡量
- 待办是否可执行
- 交付标准是否清晰
- 下一步是否具体
