# 3.20 ASR 开发手册（Qwen3-ASR-0.6B + vLLM，方案A）

## 目标
在前端输入框附件按钮旁新增“麦克风”按钮，点击后请求浏览器麦克风权限并录音；录音完成后上传到后端，后端调用自部署的 Qwen3-ASR-0.6B（vLLM）进行语音转文字，并将识别结果**追加**写入输入框。

---

## 架构概览
1. 前端录音与 UI 控制  
2. 后端转写 API  
3. vLLM 托管 Qwen3-ASR-0.6B  
4. 权限、格式与错误处理

---

## 一、前端设计

### 1.1 交互流程
1. 用户点击“麦克风”按钮  
2. 浏览器弹出麦克风权限请求  
3. 录音开始，UI 显示“录音中 + 计时”  
4. 用户点击“停止”，上传音频  
5. UI 显示“识别中”  
6. 返回文本 → 追加写入输入框

### 1.2 状态机
- `idle`：空闲  
- `recording`：录音中  
- `transcribing`：识别中  
- `error`：失败（展示提示，可重试）

### 1.3 UI 建议
在附件按钮旁新增麦克风图标按钮  
录音中显示红点或“停止”按钮  
识别中禁用再次录音

### 1.4 录音方案
使用 `MediaRecorder`  
优先 `audio/webm;codecs=opus`  
fallback `audio/wav`（部分浏览器）

### 1.5 文本写入策略
识别结果**追加**写入输入框  
前端合并方式：`inputValue = inputValue + (空格或换行) + asrText`

---

## 二、后端设计

### 2.1 新增 API
`POST /api/asr/transcribe`

请求  
- `multipart/form-data`  
- `file`：音频文件  
- `lang`：可选，例如 `zh`  
- `prompt`：可选，用于热词或上下文提示

响应  
```json
{
  "text": "识别结果文本",
  "duration_ms": 1234,
  "model": "Qwen3-ASR-0.6B"
}
```

### 2.2 校验与限制
- 文件大小限制，例如 10MB  
- 录音时长限制，例如 60s  
- 仅接受 `audio/*`  
- 超限返回 413 或 400

### 2.3 失败处理
统一返回：
```json
{
  "error": "transcribe_failed",
  "message": "详细原因"
}
```

## 三、vLLM 部署方案

### 3.1 模型准备
Qwen3-ASR-0.6B 本地/私有仓库  
确保模型文件完整（config、tokenizer、权重）

### 3.2 vLLM 启动（示例）
```bash
vllm serve Qwen3-ASR-0.6B \
  --host 0.0.0.0 \
  --port 8000 \
  --gpu-memory-utilization 0.9 \
  --max-model-len 32768
```

如有多卡可加：
```bash
--tensor-parallel-size 2
```

### 3.3 后端调用方式
后端通过 HTTP 请求 vLLM ASR 接口  
若 vLLM 兼容 OpenAI 风格接口，则复用既有 client  
否则在后端封装请求格式

---

## 四、后端到 vLLM 的适配

### 4.1 数据准备
- 将音频文件读入内存或临时路径  
- 如果 vLLM 需要 base64，则先编码  
- 统一采样率，例如 16kHz（必要时做重采样）

### 4.2 请求参数建议
- `language: "zh"`  
- `temperature: 0`  
- `prompt`: 为空或包含行业词汇（可选）

---

## 五、权限与安全

### 5.1 浏览器权限
第一次点击触发 `getUserMedia`  
拒绝时提示用户在浏览器设置中开启麦克风

### 5.2 服务安全
转写接口需要鉴权（沿用现有 Token 机制）

### 5.3 隐私
录音仅用于即时识别  
不长期保存音频（除非明确需求）

---

## 六、性能与体验建议
- 单次录音建议 15-30 秒  
- 录音完成后立即上传  
- 显示“识别中”状态，避免用户重复点击  
- 返回结果后自动聚焦输入框

---

1. 前端新增麦克风按钮

4. 连接 vLLM 服务并完成请求适配  
5. 加入参数校验与错误提示

1. 浏览器权限弹窗正常  
2. 录音开始/停止按钮状态正确  
3. 上传后可获取识别结果  
4. 识别结果正确追加到输入框
